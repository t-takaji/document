### ✅誤解が生まれない周知文章作成プロンプト
```
以下の内容を、「誰が読んでも同じ解釈になる」社内お知らせ文に変換してください。
【伝えたい内容(箇条書き・口語でOK)】
(ここにメモをそのまま貼り付け)

【配信先・読者層】
(ここにメモをそのまま貼り付け)

【過去に発生した誤解やトラブル(あれば)】
(ここにメモをそのまま貼り付け)

【出力ルール】
1. 構成:【何が変わるか】→【誰が対象か(対象外も明記)】→【いつから】→【具体的に何をすればいいか】→【例外・経過措置】→【問い合わせ先】
2. 各セクション冒頭に【 】で1行要約を入れる(流し読みでも要点が伝わるように)
3. 「〜の場合もあります」「〜を検討中です」等の曖昧表現は禁止。未決事項は「○月○日までに追って連絡します」と期限付きで明記
4. 想定FAQ(よくある誤解3つ)を末尾に追加
5. 本文は400字以内(FAQ除く)
6. 最後に「この文面で誤解が生じる可能性がある箇所」を3つ指摘し、改善案を提示
```

### ✅会議の議事録を"使える形"でまとめるプロンプト
```
✅会議の議事録を"使える形"でまとめるプロンプト
以下の会議メモを「決定事項」「未決事項」「ToDo一覧」に構造化し、
最後に関係者別のアクションサマリーを生成してください。

【会議メモ(走り書き・箇条書きそのままでOK)】
（ここに議事録・メモを貼り付け）

【参加者と役割】
（ここに参加者名と役割のメモを貼り付け）

【会議の目的・アジェンダ】
（ここに会議目的・アジェンダのメモを貼り付け）

【出力形式】
## 決定事項
- 決定内容(根拠を1行で)
## 未決事項(次回持ち越し)
- 未決内容 → 決定に必要な情報・アクション → 期限
## ToDo一覧
| 担当者 | タスク | 期限 | 成果物 | 依存関係 |
※ 会議日から逆算して現実的な期限を仮設定
※ 「検討する」「確認する」等の曖昧タスクは禁止。
必ず具体的な成果物に変換(例:「A案とB案の比較表を作成」「見積3社分を取得」)
## 関係者別アクションサマリー
(そのままSlack/メールで各担当者に転送できる形式。自分のタスクだけを抜粋した文面を人数分生成)
```

### ✅メールのストレスを無くすプロンプト
```
あなたはビジネスメールの専門家です。
以下の状況に基づき、相手との関係性を損なわず、かつこちらの要望を通すメールを作成してください。

【状況】
相手：（相手の役職・関係性）
経緯：（これまでの経緯）
伝えたいこと：（交渉／催促／お詫びの核心）
変更の理由：（なぜ変更が必要か）
・相手が気にしそうなこと:（相手の懸念）
・トーン:（丁寧/ややカジュアル/フォーマル）

【出力ルール】
1. 件名を3パターン提示(「開きたくなる件名」を意識)
2. 本文の構成は「結論→背景・経緯→具体的なお願い→相手への配慮・フォロー」の順
3. 相手の懸念を先回りしてケアする一文を必ず含める
4. 「申し訳ございません」の連発は禁止。代わりに「具体的な代替案」や「相手にとってのメリット」を提示する
5. 文字数は300〜400字。簡潔に、しかし冷たくならないバランスで
6. 送信前に自分で判断できるよう「この文面のリスクと注意点」を末尾に3つ箇条書きで付記
```
